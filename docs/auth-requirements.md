# Auth And Member Requirements

## Scope

회원가입, 로그인, 로그아웃, 인증 상태 유지, 권한 분리, 계정 상태 관리는 회원/인증 담당 영역이다. 게이미피케이션은 인증된 사용자 정보를 기반으로 캐릭터, EXP, 퀘스트, 뱃지 정보를 표시한다.

## Auth Flow Details

### Register Duplicate Check

- 회원가입 시 로그인 ID 또는 이메일 중복 여부를 확인해야 한다.
- 프론트엔드는 회원가입 제출 전 또는 입력 중 중복 확인 결과를 사용자에게 표시할 수 있어야 한다.
- 중복 확인은 최종적으로 서버 응답을 기준으로 판단한다.
- 클라이언트 검증은 사용자 편의를 위한 보조 검증이며, 서버 검증을 대체하지 않는다.

### Register Profile Fields

- 회원가입 시 사용자 이름, 생년월일, 성별을 입력받는다.
- 생년월일은 `birthDate`로 전달한다.
- 성별은 직접 텍스트 입력이 아니라 선택 버튼으로 입력받으며 값은 `female`, `male` 중 하나로 전달한다.
- 만 나이 `age`는 프론트에서 직접 입력받지 않고 백엔드에서 `birthDate` 기준으로 계산한다.
- 백엔드 저장 또는 응답 시 나이가 필요하면 계산된 `age`를 별도 필드로 제공한다.

### Login Session

- 로그인 성공 시 시스템은 사용자 인증 상태를 유지해야 한다.
- 인증 상태 유지 방식은 JWT 또는 서버 세션 중 프로젝트 구조에 맞게 선택한다.
- 인증된 사용자는 홈, 출석, 학습 기록, 게이미피케이션 화면에 접근할 수 있다.
- 비로그인 사용자가 주요 서비스에 접근하면 로그인 화면으로 이동해야 한다.

### Password Change / Reset

- 비밀번호 변경 화면은 로그인 화면과 동일한 UI 톤을 사용한다.
- 현재 프론트 화면은 다음 2단계 흐름으로 구성한다.
  1. 사용자 아이디 입력
  2. 아이디 확인 성공 후 새 비밀번호 입력
- 실제 구현 시 1단계는 사용자 존재 여부 확인 API와 연결한다.
- 실제 구현 시 2단계는 비밀번호 변경 또는 재설정 API와 연결한다.
- 보안상 사용자 존재 여부를 과도하게 노출하지 않도록 응답 메시지 정책을 별도로 정해야 한다.

## Integration With Gamification

- 로그인 성공 후 사용자의 캐릭터 커스터마이징 완료 여부를 조회해야 한다.
- 커스터마이징 미완료 사용자는 캐릭터 선택 화면으로 이동한다.
- 커스터마이징 완료 사용자는 홈으로 이동한다.
- 홈 화면의 회원 이름은 인증/회원 영역에서 제공한 표시명을 사용한다.
- 로그아웃은 단순 페이지 이동이 아니라 인증 세션 또는 토큰 무효화로 처리한다.

## User Status Model

사용자 상태는 크게 두 가지로 분리한다.

```text
1. 출결 상태
   - 하루 단위의 최종 출석 결과

2. 현재 상태
   - 사용자의 실시간 위치 또는 활동 상태
```

### Attendance Status

출결 상태는 하루 단위의 최종 출석 결과를 의미한다. 출석률, 랭킹, 관리자 출결 기록, 통계 계산에 사용된다.

| 상태 | 의미 | 예시 |
| --- | --- | --- |
| 출석 | 기준 입실 시간 이전에 입실하고, 기준 퇴실 시간 이후 퇴실한 상태 | 9시 이전 입실 + 18시 이후 퇴실 |
| 지각 | 기준 입실 시간 이후에 입실한 상태 | 9시 이후 입실 |
| 결석 | 입실 기록이 없거나 필수 입퇴실 기록이 부족한 상태 | 입실 기록 없음, 퇴실 기록 없음 |
| 조퇴 | 기준 퇴실 시간 이전에 퇴실한 상태 | 18시 이전 퇴실 |

### Current Status

현재 상태는 사용자의 실시간 위치 또는 활동 상태를 의미한다. 실습실 배치도, 관리자 대시보드, 타이머, 회의실 위치 표시, 게이미피케이션 화면에서 사용된다.

| 상태 | 의미 | 사용처 |
| --- | --- | --- |
| 재실 | 실습실 또는 지정 공간에 머무르는 상태 | 실습실 배치도, 관리자 대시보드 |
| 부재중 | 식사, 외출 등 일시적으로 자리를 비운 상태 | 공부 시간 제외, 관리자 확인 |
| 회의중 | 회의실 등 다른 공간에 있는 상태 | 위치 표시, 회의실 현재 인원 파악 |
| 퇴실 | 해당 일자의 공간 이용을 종료한 상태 | 퇴실 요약, 출결 판정 |

### Status To Review With Timer

| 상태 | 의미 | 사용처 |
| --- | --- | --- |
| 학습 중 | 학습 타이머가 실행 중인 상태 | 타이머, 공부 시간 기록 |

## Absence And Exception Reasons

부재중은 사용자가 출석한 뒤 일시적으로 자리를 비운 상태를 의미한다. 휴가, 병가, 공가 등은 별도의 현재 상태로 추가하지 않고, 관리자가 출결 상태를 변경할 때 입력하는 변경 사유로 처리한다.

| 구분 | 의미 | 처리 방식 |
| --- | --- | --- |
| 휴가 | 출석하지 않은 사유 | 관리자 변경 사유 |
| 병가 | 질병 등으로 인한 사유 | 관리자 변경 사유 |
| 공가 | 공식 일정 등으로 인한 사유 | 관리자 변경 사유 |
| 진단서 제출 | 결석/조퇴를 출석으로 인정할 근거 | 관리자 변경 사유 |

사용자가 자동 판정상 결석이 되었더라도, 관리자가 병가 또는 진단서 제출 사유를 확인한 뒤 출석으로 변경할 수 있다. 이 경우 변경 이력은 출결 변경 로그에 저장한다.

## Attendance Decision Rules

| 조건 | 출결 상태 |
| --- | --- |
| 기준 시간 이전 입실 + 기준 퇴실 시간 이후 퇴실 | 출석 |
| 기준 시간 이후 입실 + 기준 퇴실 시간 이후 퇴실 | 지각 |
| 입실 기록 없음 | 결석 |
| 입실 기록은 있으나 퇴실 기록 없음 | 결석 또는 관리자 확인 필요 |
| 기준 시간 이전 입실 + 기준 퇴실 시간 이전 퇴실 | 조퇴 |
| 부재중 시간이 제한 시간을 초과한 경우 | 조퇴 또는 관리자 확인 필요 |

## Check-Out Summary

퇴실 요약은 사용자가 퇴실 처리 후 확인하는 하루 활동 요약 정보다. 게이미피케이션 기능과 연결될 경우 완료 퀘스트, 획득 EXP, 스트릭 변화도 함께 표시할 수 있다. 뱃지는 별도 조건 시스템에서 처리한다.

| 항목 | 설명 |
| --- | --- |
| 입실 시간 | 사용자가 입실 처리한 시각 |
| 퇴실 시간 | 사용자가 퇴실 처리한 시각 |
| 체류 시간 | 입실 시간부터 퇴실 시간까지의 전체 시간 |
| 학습 시간 | 학습 타이머로 기록된 실제 학습 시간 |
| 부재중 시간 | 부재중 상태로 머문 시간 |
| 출결 상태 | 출석, 지각, 결석, 조퇴 중 하나 |

## Admin Attendance Change

관리자는 사용자의 출결 상태를 수동으로 변경할 수 있다. 변경 시에는 다음 정보를 기록해야 한다.

| 항목 | 설명 |
| --- | --- |
| 변경 대상 | 변경되는 출결 기록 |
| 변경 전 상태 | 기존 출결 상태 |
| 변경 후 상태 | 변경된 출결 상태 |
| 변경 관리자 | 변경을 수행한 관리자 |
| 변경 시간 | 변경이 발생한 시각 |
| 변경 사유 | 휴가, 병가, 공가, 진단서 제출, 기타 사유 등 |

## API Candidates

최종 API가 아니라 기능 흐름을 기준으로 필요한 API 후보를 정리한다.

| API 후보 | 설명 |
| --- | --- |
| `POST /attendances/check-in` | 입실 / 출석 체크 처리 |
| `POST /attendances/check-out` | 퇴실 처리 |
| `GET /attendances/me` | 내 출석 기록 조회 |
| `GET /attendances/me/summary` | 내 출석률 및 요약 조회 |
| `GET /admin/attendances` | 관리자 출결 기록 조회 |
| `PATCH /admin/attendances/{attendanceId}/status` | 관리자 출결 상태 수동 변경 |
| `GET /users/me/status` | 내 현재 상태 조회 |
| `PATCH /users/me/status` | 내 현재 상태 변경 |
| `GET /admin/users/status` | 관리자 사용자 현재 상태 조회 |
| `GET /auth/check-login-id?loginId={loginId}` | 로그인 ID 중복 또는 존재 여부 확인 |
| `POST /auth/login` | 로그인 처리 및 인증 상태 발급 |
| `POST /auth/logout` | 로그아웃 및 인증 상태 무효화 |
| `POST /auth/password/change` | 로그인 상태에서 기존 비밀번호 확인 후 비밀번호 변경 |
| `POST /auth/password/reset/request` | 비로그인 상태에서 아이디 또는 이메일 기반 비밀번호 재설정 요청 |
| `POST /auth/password/reset/confirm` | 재설정 검증 후 새 비밀번호 저장 |

## ERD Entity Candidates

최종 ERD가 아니라 사용자/출석 기능을 기준으로 필요한 엔티티 후보를 정리한다.

| 엔티티 | 역할 |
| --- | --- |
| `User` | 사용자 기본 정보 |
| `AuthSession` 또는 `RefreshToken` | 로그인 인증 상태 관리 |
| `Attendance` | 하루 단위 입실/퇴실 및 출결 기록 |
| `UserStatus` | 사용자의 현재 상태 |
| `AttendanceChangeLog` | 관리자 출결 상태 변경 이력 |
| `Space` | 실습실, 회의실 등 공간 정보 |
| `StudySession` | 학습 타이머 기록 |
