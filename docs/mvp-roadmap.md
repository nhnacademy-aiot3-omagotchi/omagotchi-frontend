# MVP Roadmap

## MVP Goal

MVP 목표는 사용자가 로그인 후 캐릭터를 선택하고, 홈에서 자신의 캐릭터, 레벨, EXP, 오늘의 퀘스트를 확인한 뒤 실습실로 이동할 수 있는 흐름을 완성하는 것이다.

## MVP Scope

### Included

- 회원가입/로그인/로그아웃 기본 흐름
- 인증 상태 유지
- 일반 사용자/관리자 권한 분리
- 캐릭터 선택 및 색상 선택
- 커스터마이징 완료 여부에 따른 진입 분기
- 오마고치 홈
- 오늘의 퀘스트 표시
- 실습실 이동
- 출석 처리
- 현재 상태 표시
- EXP/레벨 표시
- 포인트 지급 기록
- 기본 랭킹 조회
- 기본 뱃지 조회

### Excluded

- 상점
- 보관함
- 장비
- 펫
- 개인화 퀘스트 자동 생성
- 다중 인증
- 의심 로그인 탐지
- 고급 관리자 감사 화면

## Frontend MVP Checklist

- [ ] 로그인 화면 Bootstrap 또는 기존 폼 기준 정리
- [ ] 캐릭터 선택 화면 Custom CSS 유지
- [ ] 홈 화면 Custom CSS 유지
- [ ] 홈 화면 데이터 주입 지점 분리

- [ ] 오늘의 퀘스트를 데이터 기반으로 렌더링
- [ ] 햄버거 메뉴 항목별 라우트 또는 핸들러 연결

- [ ] 출결 상태와 현재 상태의 UI 표현 분리

## Key Decisions

- 관리자 화면은 Bootstrap 기반으로 빠르게 구성한다.
- 사용자 화면은 Custom CSS 기반으로 게임형 UI를 유지한다.
- Figma는 최종 시안이 아니라 화면 배치와 흐름을 위한 프로토타입으로 본다.
- CSS 추가 고도화는 현재 보류한다.
- 뱃지는 퀘스트 보상에 직접 포함하지 않는다.
- 최대 레벨은 30이다.
- 30레벨 이후 EXP는 누적하지만 레벨은 상승하지 않는다.
